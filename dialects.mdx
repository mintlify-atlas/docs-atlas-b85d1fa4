Sequelize supports nine SQL databases through separate dialect packages. Install `@sequelize/core` plus the package for your database, then pass the dialect class to the `Sequelize` constructor.

<Tabs>

<Tab title="PostgreSQL">

**Package:** `@sequelize/postgres`

<CodeGroup>

```bash npm
npm install @sequelize/core @sequelize/postgres
```

```bash yarn
yarn add @sequelize/core @sequelize/postgres
```

```bash pnpm
pnpm add @sequelize/core @sequelize/postgres
```

</CodeGroup>

```typescript
import { Sequelize } from '@sequelize/core';
import { PostgresDialect } from '@sequelize/postgres';

const sequelize = new Sequelize({
  dialect: PostgresDialect,
  database: 'my_database',
  user: 'my_user',
  password: 'my_password',
  host: 'localhost',
  port: 5432,
});
```

You can also pass a connection URL:

```typescript
const sequelize = new Sequelize({
  dialect: PostgresDialect,
  url: 'postgres://my_user:my_password@localhost:5432/my_database',
});
```

<Tip>
  PostgreSQL supports connecting over a UNIX socket by passing an absolute directory path as the `host` option (e.g. `host: '/var/run/postgresql'`).
</Tip>

</Tab>

<Tab title="SQLite">

**Package:** `@sequelize/sqlite3`

<CodeGroup>

```bash npm
npm install @sequelize/core @sequelize/sqlite3
```

```bash yarn
yarn add @sequelize/core @sequelize/sqlite3
```

```bash pnpm
pnpm add @sequelize/core @sequelize/sqlite3
```

</CodeGroup>

```typescript
import { Sequelize } from '@sequelize/core';
import { SqliteDialect } from '@sequelize/sqlite3';

const sequelize = new Sequelize({
  dialect: SqliteDialect,
  storage: 'path/to/my_database.sqlite',
});
```

<Note>
  SQLite uses a file path via the `storage` option instead of `host` and `port`. Use `storage: ':memory:'` for an in-memory database, which is useful for tests. In-memory databases require the pool to be limited to a single connection (`pool.max: 1`) so every query shares the same in-memory state.
</Note>

```typescript
// In-memory database for tests
const sequelize = new Sequelize({
  dialect: SqliteDialect,
  storage: ':memory:',
  pool: { max: 1 },
});
```

  },
  server: 'localhost',
  port: 1433,
});
```

<Note>
  The `@sequelize/mssql` package uses `tedious` as its underlying driver. Authentication is configured through the `authentication` object rather than top-level `user` and `password` options.
</Note>

import { Sequelize } from '@sequelize/core';

```

<Note>
  IBM i connections are established over ODBC. You must configure an ODBC DSN on the host before connecting. The `@sequelize/db2-ibmi` package depends on the `odbc` npm package, which in turn requires an ODBC driver manager and the IBM i Access ODBC driver.
</Note>

## Choosing between dialects

All dialects share the same Sequelize API. The dialect package only affects connection setup, data type mapping, and any database-specific SQL features. You can write model definitions and queries that work across dialects by sticking to the common subset of `DataTypes` and avoiding dialect-specific raw SQL.
